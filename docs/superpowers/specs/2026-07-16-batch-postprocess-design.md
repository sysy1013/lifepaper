# Phase 3: 일괄 모드 후처리 (수정본·품질 진단·오탈자) 설계

날짜: 2026-07-16 / 상태: 승인됨 (전체 페이즈 일괄 승인)

## 목적

단일 모드에만 있는 수정본 생성·품질 진단·오탈자 검사를 일괄 모드에도 제공.
교사가 반 전체를 한 번에 후처리할 수 있게 한다.

## 설계

### 공통: 병렬 실행 헬퍼

`run_parallel(n, job, label)` — ThreadPoolExecutor(BATCH_WORKERS) + 진행바 +
순서 유지. 기존 일괄 검토/초안 루프도 이 헬퍼로 리팩터링 (중복 제거).

### 모드 1 (일괄 검토) — "3️⃣ 일괄 후처리" 섹션

요약표 아래, 학생별 상세 위에 버튼 3개:

1. **전체 수정본 생성** (검출 있는 학생만): rewrite_with_gemini 병렬 —
   단일 경로와 동일한 마스킹 처리. 결과는 각 학생 dict에 `revised` 저장,
   상세 expander에 수정본 + 규칙 재검 표시, 수정본 ZIP 다운로드 제공.
2. **전체 품질 진단**: assess_quality_with_gemini 병렬, `quality` 저장.
   요약표에 "품질(평균)" 열 추가, 상세에 컴팩트 렌더링
   (expander 중첩 금지 → render_quality_compact 신설).
3. **전체 오탈자 검사**: proofread_with_gemini 병렬, `proofread` 저장.
   요약표에 "오탈자" 열 추가, 상세에 교정표 + 교정 반영본.

각 작업 실패는 학생별 오류 문자열로 격리 (전체 중단 없음).
결과는 session_state["batch_review"] 항목에 병합 — rerun에도 유지.

### 모드 2 (일괄 초안) — 동일 패턴

수정본 개념이 없으므로 **품질 진단·오탈자** 2개 버튼만.
결과는 batch_drafts 항목에 저장, 학생별 expander에 표시.

## 검증

1. 로직 검증: 스텁으로 run_parallel 순서·오류 격리 확인.
2. 실 API: 3건 일괄 검토 → 수정본·품질·오탈자 일괄 실행 스크립트 검증.
3. 단일 경로 회귀: 기존 테스트 재실행.
